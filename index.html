📖 Lecture 1: Review of CYBS 3301 – Cybersecurity Foundation

1. Introduction to Cybersecurity – مقدمة في الأمن السيبراني

▼
📌 Original (English)
Information Security: Protecting information and information systems from unauthorized access, use, disclosure, disruption, modification, or destruction.

Models for Discussing Security Issues:
• CIA Triad: Confidentiality, Integrity, Availability
• Parkerian Hexad: Adds Authenticity, Utility, Possession to CIA

Defense in Depth: A layered security strategy using multiple controls (Data → Application → Host → Internal Network → External Network).
🌐 الترجمة (Arabic)
أمن المعلومات: حماية المعلومات وأنظمة المعلومات من الوصول غير المصرح به، الاستخدام، الإفصاح، الإخلال، التعديل، أو الإتلاف.

نماذج مناقشة مسائل الأمن:
• ثالوث CIA: السرية، النزاهة، التوافر
• Parkerian Hexad: يضيف المصداقية، الفائدة، الحيازة

الدفاع بالعمق: استراتيجية أمنية متعددة الطبقات (بيانات ← تطبيق ← مضيف ← شبكة داخلية ← شبكة خارجية).
💡 الشرح (Explanation)
ثالوث CIA هو الأساس:
— C = Confidentiality (السرية): فقط من له صلاحية يرى البيانات → هجومها: Interception (الاعتراض)
— I = Integrity (النزاهة): البيانات لا تُعدَّل دون إذن → هجومها: Modification (التعديل)
— A = Availability (التوافر): النظام متاح للمستخدمين المصرح لهم → هجومها: Interruption (الانقطاع)

Parkerian Hexad يضيف 3 عناصر: Authenticity (التحقق من الهوية) + Possession (الحيازة/السيطرة) + Utility (الفائدة/الجدوى)

Defense in Depth: تخيّل الدفاع كبصلة بطبقات – إذا اخترق المهاجم طبقة، واجهته طبقة أخرى.

🎯 أسئلة تطبيقية

نقاطك: 0 إجمالي: 5

Q1. Which CIA component is violated when an attacker intercepts network traffic?
(أي مكوّن في CIA يُنتهَك عند اعتراض حركة الشبكة؟)

✅ صحيح! الاعتراض (Interception) ينتهك السرية (Confidentiality) لأن طرفاً غير مصرح له يقرأ البيانات.

Q2. True or False: The Parkerian Hexad contains only 3 elements.
(صح أم خطأ: يحتوي Parkerian Hexad على 3 عناصر فقط.)

✅ False – يحتوي على 6 عناصر: C + I + A + Authenticity + Possession + Utility.

Q3. Fill in the blank: Defense in Depth is a _____ security strategy.
(أكمل: الدفاع بالعمق هو استراتيجية أمنية _____)


Q4. Which attack type violates Integrity?
(أي نوع هجوم ينتهك النزاهة؟)

✅ صحيح! التعديل (Modification) ينتهك النزاهة لأنه يغير البيانات دون إذن.

Q5. The outermost layer in Defense in Depth is:
(الطبقة الخارجية في الدفاع بالعمق هي:)

✅ صحيح! الشبكة الخارجية هي الطبقة الأبعد، والبيانات هي الطبقة الداخلية (المركز).

2. Identification & Authentication – التعريف والمصادقة

▼
📌 Original (English)
Identification: An assertion of who we are.
Authentication: A set of methods used to establish whether a claim of identity is true.

Common Authentication Methods:
• Passwords – Something you know
• Biometrics – Something you are
• Hardware Tokens – Something you have
🌐 الترجمة (Arabic)
التعريف (Identification): إثبات هويتنا – أي "من أنا".
المصادقة (Authentication): مجموعة من الأساليب للتحقق من صحة ادعاء الهوية.

أساليب المصادقة الشائعة:
• كلمات المرور – شيء تعرفه
• القياسات الحيوية – شيء أنت عليه
• الرموز المادية – شيء تمتلكه
💡 الشرح (Explanation)
الفرق المهم: Identification = ادّعاء الهوية (مثل كتابة اسم المستخدم)
Authentication = إثبات الهوية (مثل إدخال كلمة المرور).

الـ 3 عوامل: Something you Know (كلمة مرور) + Something you Have (token) + Something you Are (بصمة) — استخدام اثنين منها = Multi-Factor Authentication (MFA).

🎯 أسئلة تطبيقية

Q1. True or False: Identification and Authentication are the same process.
(صح أم خطأ: التعريف والمصادقة عملية واحدة.)

✅ False – Identification = ادعاء الهوية، Authentication = التحقق من صحة الادعاء.

Q2. Biometrics falls under which authentication factor?
(القياسات الحيوية تندرج تحت أي عامل مصادقة؟)

✅ صحيح! القياسات الحيوية (بصمة، وجه) = "شيء أنت عليه" = Something you are.

Q3. A hardware token is classified as:
(رمز المصادقة المادي يُصنَّف كـ:)

✅ صحيح! الرمز المادي = "شيء تمتلكه" = Something you have.

3. Authorization & Access Control – التفويض والتحكم بالوصول

▼
📌 Original (English)
Access Control: Giving a party access to a given resource.

Implementation methods:
• Access Control Lists (ACLs)
• Capabilities

Common Access Control Models:
1. DAC – Discretionary Access Control
2. MAC – Mandatory Access Control
3. RBAC – Role-Based Access Control
4. Rule-Based Access Control
5. ABAC – Attribute-Based Access Control
6. Multi-Level Access Control (Bell–LaPadula Model and Biba Model)

Principle of Least Privilege: Users should have only the minimum access needed.
🌐 الترجمة (Arabic)
التحكم بالوصول: منح طرف ما صلاحية الوصول إلى مورد معين.

نماذج التحكم بالوصول:
1. DAC – التحكم التقديري: يحدد مالك المورد الصلاحيات
2. MAC – التحكم الإلزامي: تضعه الإدارة، لا يمكن تغييره
3. RBAC – التحكم القائم على الأدوار: الصلاحيات حسب الوظيفة
4. Rule-Based – قواعد محددة مسبقاً
5. ABAC – التحكم القائم على الصفات
6. Multi-Level – نماذج Bell-LaPadula وBiba

مبدأ أقل صلاحية: يحصل المستخدم فقط على الحد الأدنى من الصلاحيات الضرورية.
💡 الشرح (Explanation)
Bell–LaPadula Model: يركز على السرية (Confidentiality) — "No Read Up, No Write Down" (لا تقرأ من مستوى أعلى، لا تكتب لمستوى أدنى)

Biba Model: يركز على النزاهة (Integrity) — عكس Bell-LaPadula

Physical Access Control: التحكم في دخول وخروج الأفراد من المباني.

🎯 أسئلة تطبيقية

Q1. Which access control model is based on job roles?
(أي نموذج تحكم بالوصول يعتمد على الوظائف؟)

✅ RBAC = Role-Based Access Control = الصلاحيات تُحدَّد حسب دور/وظيفة المستخدم.

Q2. The Bell–LaPadula model focuses on protecting:
(نموذج Bell–LaPadula يركز على حماية:)

✅ Bell–LaPadula = السرية (Confidentiality) | Biba = النزاهة (Integrity).

Q3. True or False: The Principle of Least Privilege means giving users maximum access to do their jobs efficiently.
(صح أم خطأ: مبدأ أقل صلاحية يعني منح المستخدمين أقصى صلاحيات لأداء عملهم بكفاءة.)

✅ False – مبدأ أقل صلاحية يعني منح الحد الأدنى من الصلاحيات الضرورية فقط.

4. Cryptography Basics – أساسيات التشفير

▼
📌 Original (English)
Cryptography: The science of protecting the confidentiality and integrity of data. Depends on difficult mathematical problems (one-way problems).

Types:
• Symmetric Cryptography: Same key for encryption and decryption (e.g. AES)
• Asymmetric Cryptography: Public key encrypts, private key decrypts (e.g. RSA)
• Hash Functions: One-way transformation, fixed output
• Digital Signatures: Verify authenticity and non-repudiation
• Digital Certificates: Bind identity to a public key

Data Protection states: At Rest, in Motion, in Use
🌐 الترجمة (Arabic)
التشفير: علم حماية سرية ونزاهة البيانات. يعتمد على مسائل رياضية صعبة أحادية الاتجاه.

الأنواع:
• التشفير المتماثل: نفس المفتاح للتشفير وفك التشفير (AES)
• التشفير غير المتماثل: مفتاح عام للتشفير، مفتاح خاص لفك التشفير (RSA)
• دوال التجزئة: تحويل أحادي الاتجاه، مخرج ثابت الحجم
• التوقيعات الرقمية: التحقق من المصداقية وعدم الإنكار
• الشهادات الرقمية: ربط الهوية بمفتاح عام

حالات البيانات: في حالة السكون، أثناء النقل، أثناء الاستخدام
💡 الشرح (Explanation)
الفرق بين Symmetric و Asymmetric:
— Symmetric: مفتاح واحد → أسرع، لكن مشكلة توزيع المفتاح
— Asymmetric: مفتاحان (عام + خاص) → أبطأ، لكن أكثر أماناً لتبادل المفاتيح

Hash = بصمة رقمية: MD5, SHA-256 — لا يمكن عكسها → تُستخدم للتحقق من سلامة الملفات.

Digital Signature: يضمن عدم الإنكار (Non-repudiation) — لا تستطيع أن تقول "لم أرسل هذا".

🎯 أسئلة تطبيقية

Q1. Which cryptography type uses a single key for both encryption and decryption?
(أي نوع تشفير يستخدم مفتاحاً واحداً؟)

✅ التشفير المتماثل (Symmetric) يستخدم نفس المفتاح لتشفير وفك تشفير البيانات.

Q2. True or False: Hash functions can be reversed to retrieve the original data.
(صح أم خطأ: يمكن عكس دوال التجزئة لاسترجاع البيانات الأصلية.)

✅ False – دوال التجزئة أحادية الاتجاه (one-way) لا يمكن عكسها.

Q3. Digital signatures provide which security property?
(التوقيعات الرقمية توفر أي خاصية أمنية؟)

✅ التوقيعات الرقمية تضمن عدم الإنكار (Non-repudiation) — لا يمكن إنكار إرسال الرسالة.

5. Governance & Cyber Risk Management – الحوكمة وإدارة مخاطر الأمن

▼
📌 Original (English)
5-Step Risk Management Process:
Step 1: Identification of Critical Information
Step 2: Analysis of Threats
Step 3: Analysis of Vulnerabilities
Step 4: Assessment of Risks
Step 5: Application of Countermeasures
🌐 الترجمة (Arabic)
عملية إدارة المخاطر – 5 خطوات:
الخطوة 1: تحديد المعلومات الحيوية
الخطوة 2: تحليل التهديدات
الخطوة 3: تحليل الثغرات
الخطوة 4: تقييم المخاطر
الخطوة 5: تطبيق التدابير المضادة
💡 الشرح (Explanation)
تذكّر: I-T-V-R-C (Information → Threats → Vulnerabilities → Risks → Countermeasures)

— Vulnerability: ثغرة في النظام (نقطة ضعف)
— Threat: مهاجم يستطيع استغلال الثغرة
— Risk = التقاء الثغرة + التهديد + احتمالية + تأثير

🎯 أسئلة تطبيقية

Q1. What is the correct order of the risk management steps?
(ما الترتيب الصحيح لخطوات إدارة المخاطر؟)

✅ الترتيب: (1) تحديد المعلومات ← (2) تهديدات ← (3) ثغرات ← (4) مخاطر ← (5) تدابير مضادة.

Q2. Fill in the blank: Risk is a convergence of a _____ and a threat.
(أكمل: المخاطرة هي تقاطع _____ وتهديد.)


6. Protecting Data, Systems & Networks – حماية البيانات والأنظمة والشبكات

▼
📌 Original (English)
Physical Threats: Fire, flood, theft, vandalism, unauthorized access to hardware.

Physical Security Controls: Fences, locks, guards, surveillance cameras, access badges, biometric readers, mantraps.

Three areas of protection:
• Protecting People: Safety procedures, emergency response
• Protecting Data: Backups, encryption, access controls
• Protecting Equipment: Cabling, power supplies, hardware locks
🌐 الترجمة (Arabic)
التهديدات المادية: الحريق، الفيضان، السرقة، التخريب، الوصول غير المصرح به للأجهزة.

ضوابط الأمن المادي: الأسوار، الأقفال، الحراسة، كاميرات المراقبة، بطاقات الوصول، أجهزة القياس الحيوي، غرف الوصول المزدوج.

ثلاثة مجالات للحماية:
• حماية الأفراد: إجراءات السلامة، الاستجابة لحالات الطوارئ
• حماية البيانات: النسخ الاحتياطي، التشفير، ضوابط الوصول
• حماية المعدات: الكابلات، مصادر الطاقة، أقفال الأجهزة
💡 الشرح (Explanation)
الأمن المادي هو الطبقة الأساسية قبل أي أمن رقمي — لأنه بدون أمن مادي، يمكن للمهاجم الوصول الجسدي للأجهزة وتجاوز كل الضمانات الرقمية.

Mantrap (فخ الدخول المزدوج): غرفة صغيرة بين بابين لا يُفتح الثاني إلا بعد إغلاق الأول وتأكيد الهوية.

🎯 أسئلة تطبيقية

Q1. Physical security controls include: (ضوابط الأمن المادي تشمل:)

✅ الأمن المادي = أسوار، أقفال، كاميرات مراقبة، بطاقات وصول، حراسة — وليس أدوات رقمية.

Q2. True or False: Physical security is less important than digital security since most attacks happen online.
(صح أم خطأ: الأمن المادي أقل أهمية من الأمن الرقمي.)

✅ False – الأمن المادي أساسي ومكمّل للأمن الرقمي؛ الوصول الجسدي يتيح تجاوز كل الضمانات الرقمية.

7. Protecting Networks – حماية الشبكات

▼
📌 Original (English)
Two methods to protect networks:
Method 1 – Secure Design: Design networks so they are resistant to attack or technical incidents. (e.g., network segmentation, DMZ, VLANs)

Method 2 – Implement Devices: Deploy security devices in and around networks:
• Firewalls: Filter traffic based on rules — control what enters/leaves the network
• Intrusion Detection Systems (IDS): Monitor and alert on suspicious activity
• Intrusion Prevention Systems (IPS): Detect AND actively block attacks
🌐 الترجمة (Arabic)
طريقتان لحماية الشبكات:
الطريقة 1 – التصميم الآمن: تصميم الشبكات لتكون مقاومة للهجمات (تجزئة الشبكة، DMZ، VLAN)

الطريقة 2 – تنفيذ الأجهزة:
• الجدران النارية (Firewalls): تصفية حركة المرور وفق قواعد محددة
• أنظمة كشف التسلل (IDS): المراقبة والتنبيه عند نشاط مشبوه
• أنظمة منع التسلل (IPS): الكشف والتصدي الفعّال للهجمات
💡 الشرح (Explanation)
الفرق بين IDS و IPS:
— IDS (Detection) = يراقب ويُنبّه فقط — مثل كاميرا مراقبة
— IPS (Prevention) = يراقب ويمنع — مثل حارس أمني

Firewall: خط الدفاع الأول — يفحص الحزم الواردة والصادرة ويسمح أو يمنع وفق سياسات محددة.

🎯 أسئلة تطبيقية

Q1. Which device monitors suspicious network activity and sends alerts but does NOT block attacks?
(أي جهاز يراقب النشاط المشبوه وينبّه لكنه لا يمنع الهجمات؟)

✅ IDS = يكشف وينبّه فقط. IPS = يكشف ويمنع فعلياً. الجدار الناري = يصفّي الحركة.

Q2. True or False: One method to protect networks is to design them so they are resistant to attack.
(صح أم خطأ: إحدى طرق حماية الشبكات هي تصميمها لتكون مقاومة للهجمات.)

✅ True – الطريقة الأولى هي التصميم الآمن للشبكة (مقاومة للهجمات)، والثانية تنفيذ أجهزة أمنية.

8. Detecting & Responding to Cyber Incidents – الكشف والاستجابة للحوادث

▼
📌 Original (English)
OS Hardening (6 actions):
1. Remove unnecessary software
2. Remove unneeded services
3. Alter default accounts
4. Use principles of least privilege
5. Perform updates
6. Implement logging and auditing

Protecting Against Malware:
• Use Anti-malware Tools

OS Security Tools:
• Scanners: Examine how your hosts interact with the rest of the devices on the network
• Vulnerability Assessment Tools: Help point out particular areas that might contain applications or services vulnerable to attack
🌐 الترجمة (Arabic)
تصلب نظام التشغيل – 6 إجراءات:
1. إزالة البرامج غير الضرورية

3. تغيير الحسابات الافتراضية
4. تطبيق مبدأ أقل صلاحية
5. إجراء التحديثات
6. تنفيذ التسجيل والتدقيق

أدوات أمن نظام التشغيل:

• أدوات تقييم الثغرات: تشير إلى المناطق التي تحتوي على تطبيقات أو خدمات ضعيفة

✅ منح جميع المستخدمين صلاحيات المدير يتعارض مع مبدأ أقل صلاحية — فهو خطر وليس تصلبًا.

(أدوات أمن نظام التشغيل تتضمن ماسحات تفحص:)

9. Technologies & Solutions – التقنيات والحلول (Mobile, Embedded, IoT)

▼
📌 Original (English)
Mobile Security:
• MDM – Mobile Device Management: centrally manage and secure mobile devices
• Common issues: Jailbreaking/Rooting (remove restrictions), Malicious Apps, Updates

Embedded Security: Securing devices with embedded systems:
• Industrial Control Systems (ICS)
• Medical Devices
• Cars

Internet of Things (IoT) Security: Securing networked "smart" devices:
• Printers, Cameras, Security devices
• Smart appliances: Refrigerators, smart bulbs, coffee machines, etc.
🌐 الترجمة (Arabic)

1. Introduction to Cyber Threats – مقدمة في التهديدات الإلكترونية

▼

Key Definitions:

• Threat: An attacker who can access the flaw and has the capability to exploit it
• Risk: A convergence of a vulnerability and threat that has a defined likelihood and impact
• Vulnerability Management: The cyclical practice of identifying, classifying, remediating, and mitigating vulnerabilities

4. Social Engineering Lifecycle – دورة حياة الهندسة الاجتماعية

▼
📌 Original (English)
Social Engineering Lifecycle (4 stages):
1. Investigate: Identify victim, gather background info, select attack method
2. Hook: Engaging the target, spinning a story, taking control of the interaction
3. Play: Expanding the foothold, executing the attack, disrupting business or siphoning data
4. Exit: Removing all traces of malware, covering tracks, bringing the play to a natural end

How to Identify Social Engineering Attacks (3+ traits = suspicious):
1. Message is unexpected
2. Request is out of the ordinary
3. Message includes a sense of urgency
4. Request could cause harm
5. Communication includes an unusual file or URL

Quid Pro Quo: Attacker offers a service in exchange for information (e.g., fake security update in exchange for disabling AV).
🌐 الترجمة (Arabic)
دورة حياة الهندسة الاجتماعية – 4 مراحل:
1. التحقيق: تحديد الضحية، جمع المعلومات، اختيار طريقة الهجوم
2. الإيقاع: إشراك الهدف، نسج قصة، التحكم في التفاعل
3. التنفيذ: توسيع قدم الهجوم، تنفيذه، تعطيل العمل أو سحب البيانات
4. الخروج: إزالة آثار البرامج الضارة، تغطية المسار، إنهاء الهجوم بشكل طبيعي

كيف تتعرف على هجمات الهندسة الاجتماعية:
3 أو أكثر من العلامات التالية = احذر!
1. الرسالة غير متوقعة
2. الطلب غير اعتيادي
3. إحساس بالإلحاح
4. الطلب قد يسبب ضرراً
5. ملف أو رابط غريب
💡 الشرح (Explanation)
تذكّر الـ 4 مراحل: I → H → P → E (Investigate → Hook → Play → Exit)

Quid Pro Quo (اللاتينية: "شيء مقابل شيء") — مثال: "سأحدّث برنامج الأمان لديك مجاناً، فقط أوقف برنامج الحماية!" ← ثم يثبت البرنامج الضار بدلاً من التحديث.

Baiting: إغراء بعروض مجانية (فيلم، بطاقة هدايا) لجعل المستخدم ينقر على رابط يحمل برنامجاً ضاراً.

8. PASTA Model – Process for Attack Simulation and Threat Analysis

▼
📌 Original (English)
PASTA = Process for Attack Simulation and Threat Analysis
Key Characteristics: Risk-Centric, Holistic Approach, Iterative Process
A seven-step methodology:
1. Preparation
2. Application Decomposition
3. Threat Analysis (uses STRIDE + attack trees)
4. Vulnerability Analysis
5. Attack Enumeration
6. Risk Analysis
7. Risk Management
🌐 الترجمة (Arabic)
PASTA = عملية محاكاة الهجوم وتحليل التهديد – 7 خطوات:
1. الإعداد (أهداف ونطاق)
2. تفكيك التطبيق (البنية، المكونات، تدفق البيانات)
3. تحليل التهديدات (يستخدم STRIDE وأشجار الهجوم)
4. تحليل الثغرات
5. تعداد الهجمات (خريطة سيناريوهات الهجوم)
6. تحليل المخاطر (احتمالية + تأثير)
7. إدارة المخاطر (تخفيف + تحكم + تدريب)
💡 الشرح (Explanation)

يتميز بـ: مرتكز على المخاطر | شامل | تكراري | تشاركي
محدودياته: معقد | يستغرق وقتاً | يحتاج موارد كبيرة

9. VAST, TRIKE & OCTAVE Models

▼
📌 Original (English)
VAST = Vulnerability, Asset, Security, Threat
• Risk management covering the entire Software Development Lifecycle (SDLC)
• Scalable for enterprise-wide threat modeling
• Uses automated tools and visualization

TRIKE = Combines STRIDE + VAST strengths
• Defines system requirement model → Risk Assessment (CRUD) → Data Flow Diagram (DFD) → Assigning Risk Values
• CRUD = Creating, Reading, Updating, Deleting
• Open-source, focuses on security requirements

OCTAVE = Operationally Critical Threat, Asset, and Vulnerability Evaluation
• 3 Phases: Building Asset-Based Threat Profiles → Developing Security Strategies → Implementing & Maintaining Controls
• Focuses on CIA of IT systems
🌐 الترجمة (Arabic)
VAST = ثغرة، أصل، أمن، تهديد — يغطي دورة حياة تطوير البرامج كاملة — قابل للتوسع

TRIKE = يجمع مزايا STRIDE و VAST — مفتوح المصدر — يركز على متطلبات الأمن
CRUD = إنشاء، قراءة، تحديث، حذف

OCTAVE = تقييم التهديدات والأصول والثغرات التشغيلية الحرجة
3 مراحل: بناء ملفات التهديد ← تطوير الاستراتيجيات ← تنفيذ وصيانة الضوابط
💡 الشرح (Explanation)
ملخص الفروق:
— VAST: للمؤسسات الكبيرة + يعتمد على الأتمتة + يغطي SDLC كاملاً
— TRIKE: مفتوح المصدر + يجمع STRIDE وVAST + يستخدم CRUD
— OCTAVE: يبدأ من الأصول الحرجة + يركز على CIA + 3 مراحل واضحة

🎯 أسئلة تطبيقية

Q1. OCTAVE stands for:
(OCTAVE اختصار لـ:)

✅ OCTAVE = Operationally Critical Threat, Asset, and Vulnerability Evaluation.

Q2. TRIKE is described as a combination of:
(TRIKE يُوصف بأنه مزيج من:)

✅ TRIKE يجمع مزايا STRIDE (تصنيف التهديدات) و VAST (النظرة الشاملة).

Q3. VAST is designed to cover the entire:
(VAST مصمم ليغطي:)

✅ VAST مصمم لتغطية دورة حياة تطوير البرامج كاملة (SDLC).

Q4. The FIRST phase of OCTAVE is:
(المرحلة الأولى من OCTAVE هي:)

✅ OCTAVE 3 مراحل: (1) بناء ملفات التهديد المستندة إلى الأصول ← (2) تطوير الاستراتيجيات ← (3) تنفيذ الضوابط وصيانتها.

✅ True – TRIKE يستخدم CRUD (إنشاء، قراءة، تحديث، حذف) في تقييم المخاطر لتحليل عمليات البيانات.

Q6. VAST stands for: (VAST اختصار لـ:)

✅ VAST = Vulnerability, Asset, Security, Threat — منهجية إدارة المخاطر التي تغطي SDLC كاملاً.

10. Attack Trees – أشجار الهجوم

▼
📌 Original (English)
A hierarchical representation of potential attacks on a system.
• Root node: The attacker's overall goal (e.g., gain unauthorized access)
• Sub-goals: Steps to achieve the main goal
• Leaves: Specific vulnerabilities to exploit
• Each vulnerability has a probability and impact score

How Attack Trees Work (5 steps):
1. Define the Goal → 2. Break Down into Sub-goals → 3. Identify Vulnerabilities → 4. Assign Probabilities and Impact → 5. Analyze and Mitigate
🌐 الترجمة (Arabic)
تمثيل هرمي للهجمات المحتملة:
• العقدة الجذرية: الهدف الرئيسي للمهاجم
• الأهداف الفرعية: خطوات تحقيق الهدف الرئيسي
• الأوراق: ثغرات محددة للاستغلال

مثال بنكي: "سرقة المال من البنك" ← إما "من الصراف الآلي" (سرقة البطاقة أو تصفح PIN) أو "اختراق الحساب الإلكتروني"
💡 الشرح (Explanation)
تخيّل شجرة مقلوبة: الجذر في الأعلى (الهدف) → الأغصان (خطوات) → الأوراق (ثغرات).

مثال (Viruses in Typical Systems):
الجذر: "الفيروس يصيب ملفاً" → إما "يعمل كمدير" أو "يعمل كمستخدم عادي" → تحتها ثغرات أكثر تفصيلاً.

المزايا: منظم + بصري + يحدد الأولويات + يُسهل التواصل مع أصحاب المصلحة

🎯 أسئلة تطبيقية

Q1. In an Attack Tree, what does the "root node" represent?
(في شجرة الهجوم، ماذا تمثل "العقدة الجذرية"؟)

✅ العقدة الجذرية = الهدف الرئيسي الإجمالي للمهاجم (مثال: الحصول على وصول غير مصرح به).

Q2. True or False: Attack tree leaves represent the attacker's main goal.
(صح أم خطأ: أوراق شجرة الهجوم تمثل الهدف الرئيسي للمهاجم.)

✅ False – الأوراق تمثل الثغرات المحددة القابلة للاستغلال، أما الهدف الرئيسي فهو العقدة الجذرية.

11. OWASP Top Ten, MITRE ATT&CK & CVSS

▼
📌 Original (English)
OWASP Top Ten: Open Web Application Security Project list of most prevalent/dangerous web app vulnerabilities. Updated every few years.
2021 Top: (1) Broken Access Control (2) Cryptographic Failures (3) Injection (4) Insecure Design (5) Security Misconfiguration (6) Vulnerable & Outdated Components (7) Identification & Authentication Failures (8) Software & Data Integrity Failures (9) Security Logging & Monitoring Failures (10) Server-Side Request Forgery

MITRE ATT&CK: Cybersecurity framework for threat modeling, penetration testing, defense creation. Divides cyberattack lifetime into 14 stages (Tactics). US government-funded research (FFRDC).

CVSS: Common Vulnerability Scoring System — assigns numerical severity score to vulnerabilities. Maintained by FIRST. Established by NIST.
🌐 الترجمة (Arabic)
OWASP Top Ten: قائمة أكثر ثغرات تطبيقات الويب شيوعاً وخطورة. تُحدَّث كل بضع سنوات.
أول 3 في 2021: (1) التحكم المكسور بالوصول (2) إخفاقات التشفير (3) الحقن

MITRE ATT&CK: إطار الأمن السيبراني لنمذجة التهديدات واختبار الاختراق. يقسم الهجوم إلى 14 مرحلة تكتيكية. ممول حكومياً أمريكياً.

CVSS: نظام تسجيل الثغرات الشائعة — يعطي درجة رقمية للثغرات. يدير FIRST، أسسه NIST.
